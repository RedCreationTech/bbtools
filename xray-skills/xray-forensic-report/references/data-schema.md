# Data Schema

This skill reads the `data.json` generated by `xray report`.

Primary fields consumed:

- `repo.root`
- `params.since`
- `params.until`
- `params.path`
- `timeseries[]`
- `hotspots[]`
- `hotspots_dirs[]`
- `risk[]`
- `coupling_pairs[]`
- `ownership_long[]`
- `staleness[]`
- `knowledge_loss[]`
- `raw.commits[]`
- `raw.authors[]`

Optional behavior:

- If `raw` is missing, author and commit-level sections degrade gracefully.
- If `knowledge_loss` is empty, the corresponding report section falls back to a note.
- If `complexity` is sparse, risk tables still render from the available data.
